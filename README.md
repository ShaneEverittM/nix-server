# nixos — home server configuration

Flake-based [NixOS](https://nixos.org/) configuration for **`nixos`**, a home server
built from a repurposed Razer laptop. Managed declaratively; this repo is the source
of truth for the system.

## Machine

- Hostname: `nixos`
- Repurposed laptop running headless as a server (lid-switch handling is set to
  `ignore` so closing the lid doesn't suspend it).
- Disk: btrfs subvolumes for `/`, `/home`, `/nix`; separate vfat `/boot`.
- Networking: Wi-Fi via `networking.wireless` (wpa_supplicant), mDNS via Avahi
  (reachable as `nixos.local` on the LAN).
- Channel: `nixpkgs/nixos-26.05`.

## Layout

| File                         | Purpose                                                       |
| ---------------------------- | ------------------------------------------------------------- |
| `flake.nix`                  | Flake entrypoint; defines `nixosConfigurations.nixos`.        |
| `configuration.nix`          | Main system configuration.                                    |
| `hardware-configuration.nix` | Generated by `nixos-generate-config` (disks, kernel modules). |
| `.zed/settings.json`         | Project-local Zed editor settings (nixd LSP + nixfmt).        |

## Rebuilding

`nh` is configured with `NH_FLAKE` pointing at this repo, so from any shell:

```bash
nh os switch      # build + activate + set as boot default
nh os boot        # build + set as boot default, activate on next reboot
```

Equivalent without `nh`:

```bash
sudo nixos-rebuild switch --flake /home/shane/.config/nix#nixos
```

## Bootstrapping a fresh install (manual steps)

Most of the system is reproduced by the flake, but a few secrets/credentials live
**outside** version control and must be recreated by hand if the machine is wiped
and reinstalled.

### Wi-Fi PSK secret

`configuration.nix` references the network's pre-shared key indirectly:

```nix
networking.wireless.networks."Marconi".pskRaw = "ext:psk_Marconi";
networking.wireless.secretsFile = "/etc/wpa_supplicant/wireless.conf";
```

The `ext:psk_Marconi` value tells wpa_supplicant to read a variable named
`psk_Marconi` from `secretsFile`. That file is **not** in this repo (it holds a
secret) and must be created manually:

1. Compute the raw (hashed) PSK from the Wi-Fi passphrase:

    ```bash
    wpa_passphrase Marconi 'YOUR_WIFI_PASSWORD'
    ```

    Copy the `psk=` value (a 64-hex-char string) from the output.

2. Write it into the secrets file as the `psk_Marconi` variable:

    ```bash
    echo 'psk_Marconi=PASTE_THE_64_HEX_HASH_HERE' | sudo tee /etc/wpa_supplicant/wireless.conf
    ```

3. Lock down ownership and permissions (wpa_supplicant runs as its own user and
   reads the file via its group):

    ```bash
    sudo chown root:wpa_supplicant /etc/wpa_supplicant/wireless.conf
    sudo chmod 0640 /etc/wpa_supplicant/wireless.conf
    ```

    The result should look like:

    ```
    -rw-r----- 1 root wpa_supplicant  /etc/wpa_supplicant/wireless.conf
    ```

4. Rebuild (or `systemctl restart wpa_supplicant.service`) to bring Wi-Fi up.

> Note: on a fresh install from the ISO you'll need some form of connectivity
> (e.g. a temporary `wpa_cli`/`nmtui` session, or Ethernet) to fetch packages
> before this declarative config can take over.

### User login password

The `shane` account is declared in the flake, but its password is not — it's
normally set during installation. If you ever need to (re)set it:

```bash
sudo passwd shane
```

SSH access itself is declarative — the authorized public key lives in
`configuration.nix` — so key-based login works as soon as the system is rebuilt.

## Known quirks

### git: "insufficient permission for adding an object"

Running `nh os switch` / `nixos-rebuild` (as root, via sudo) against this
git-tracked flake makes Nix invoke `git` as root, which can leave some objects
under `.git/objects` owned by `root`. A later `git commit` as `shane` then fails:

```
error: insufficient permission for adding an object to repository database .git/objects
```

Fix — nothing is corrupted, it's just an ownership mismatch:

```bash
sudo chown -R shane:users /home/shane/.config/nix/.git
```

Committing changes _before_ rebuilding reduces how often this happens.

## Editor

`.zed/settings.json` configures [Zed](https://zed.dev/) to use the `nixd`
language server and `nixfmt` (format-on-save) for Nix files. Both binaries are
installed by `configuration.nix` (`environment.systemPackages`), which matters
because Zed runs them on this host over remote SSH. The Zed `nix` extension must
be installed client-side (Zed has no per-project extension declaration).
